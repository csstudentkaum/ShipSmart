ShipSmart Website — Storyboard

CPCS 403 Assignment 1 | ShipSmart Universal Shipment Tracker

Project Overview

ShipSmart is a unified shipment tracking web application that allows users to track packages across multiple carriers (Aramex, DHL, FedEx, SMSA) from a single interface. This storyboard defines the structure, flow, and content of the 5-page site required for the assignment.

Site Flow Overview

index.html → services.html → schedule.html → video.html → feedback.html

All pages share a consistent header and footer with site-wide navigation. Users can move between pages in any order via the main nav links.

Page Storyboard

1. Home

index.html

Purpose
Introduce ShipSmart and provide immediate access to the core tracking feature.
Main content
Hero section, tracking form (number + carrier), supported carrier logos, feature cards (multi-carrier tracking, progress timeline, fast UI), how-it-works steps, testimonial, contact info.
User interaction / outcome
User enters tracking number and carrier to view shipment status. Can use demo mode to see results without a backend.

2. Services

pages/services.html

Purpose
Describe the services and benefits ShipSmart offers.
Main content
Service cards, benefits grid, and feature overview (multi-carrier tracking, progress timeline, modern UI).
User interaction / outcome
User learns about available features and decides to explore tracking, schedule, or feedback.

3. Schedule

pages/schedule.html

Purpose
Display weekly pickup and delivery windows for each carrier in a structured, printable format.
Main content
Properly formatted HTML table with caption, thead, tbody, rowspan and colspan. Important notes and express delivery zones. print.css ensures only the table prints.
User interaction / outcome
User views or prints the schedule to plan shipments. Print preview shows only table content.

4. Video

pages/video.html

Purpose
Provide video demonstration or promotional content about ShipSmart.
Main content
Embedded video (demo, tutorial, or promotional) with supporting text.
User interaction / outcome
User watches the video to understand how ShipSmart works or see a demo.

5. Feedback

pages/feedback.html

Purpose
Collect user feedback to improve the service.
Main content
Structured form: Name, Email, rating (Good/Average/Poor), service checkboxes, preferred carrier dropdown, comments textarea. Client-side validation in main.js; submits to process_feedback.php.
User interaction / outcome
User fills form and submits. Validation prevents invalid submission; success message shown after submission.

User Journey / Navigation

A typical path through the site:

  1. 1 User lands on Home and learns about ShipSmart.
  2. 2 User may visit Services to understand features and benefits.
  3. 3 User checks Schedule to see pickup and delivery windows; can print the table.
  4. 4 User watches the Video for a demo or tutorial.
  5. 5 User submits Feedback to share their experience.

Navigation is non-linear: header and footer links allow access to any page at any time.

Technical Notes

  • CSS: global/main.css — table and form styling; external stylesheet used across all pages.
  • Print: global/print.css — table-only printing when user prints from Schedule page.
  • JavaScript: scripts/main.js — feedback form validation, nav toggle, tracking demo.
  • Backend: server/process_feedback.php — receives and stores feedback.
  • Structure: Minimum 5 pages; consistent header/footer; semantic HTML; HTML/CSS validation; Edge compatibility.

This storyboard supports the CPCS 403 assignment requirements: 5 pages, schedule table, feedback form, external CSS, print.css, and validation. All pages are linked via consistent navigation.
